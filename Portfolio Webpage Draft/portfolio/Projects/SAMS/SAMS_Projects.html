SAMS-tagram 3.0

For SAMS-tagram 3.0 we connected both HTML pages to Google Firebase. For the News Feed I used a similar design as the TODO List by creating a new collection cost Post saving the input values into unique and computer-generated documents. To delete a post I just got the doc id from the database and deleted the particular document and removed it from the HTML page. When you reload the page the earlier posts display using query Snapshots to get each document from the collection and add the values to the Post div.

The profile page used a different design because I named the document amin_profile and each type you input text into the form it would set the values of main_profile to be the input value. This method was needed because I wasn't creating a new document each time I type in the text but rather calling upon one document and changing the information. Also when I reload the page I couldn't have multiple documents display but I just needed the information in that one document to be displayed.

Typing Coach

This was our first basic game using JAVAscript, HTML and a little CSS. Within the JAVAscript file you had an array with a list of 20 words. Every 3 seconds a random word would be selected and displayed on the page at a random location. At the top of the page, you had a form where you could type the words that were displayed on the page. By using an if statement we were able to cross-reference whether the typed value matched any of the values displayed on the page and if it was true the word was removed. The objective of the game was to constantly keep the page clear of any words. One issue I faced was that the words would sometimes display outside the viewable screen and you would have to scroll either down or to the right to see the word.
